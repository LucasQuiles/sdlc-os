# Success Library

Safety-II successful adaptation library. Updated by LOSA Observer after each observation session.

**Purpose (Dekker Safety-II):** Traditional safety analysis (Safety-I) focuses on what goes wrong. Safety-II recognizes that the same human and agent adaptations that occasionally fail are also what makes the system succeed every day. Capturing successful adaptations builds a positive reference: "what does good look like?" This library is used during Evolve cycles to reinforce effective patterns, not just fix broken ones.

---

## Library

*Library is empty. Populated by LOSA Observer when agents correctly handle ambiguity, novel situations, or conflicting requirements.*

---

## Entry Format

When the LOSA Observer captures a successful adaptation, entries follow this format:

```
### [task-ID] — [brief description]

**Date:** YYYY-MM-DD
**Agent:** [agent name]
**Situation:** [what was ambiguous, novel, or conflicting]
**Adaptation:** [what the agent did]
**Why it worked:** [why this was effective — not just what happened]
**Generalizable pattern:** [can this be extracted as a convention or constitution rule?]
```

---

## Usage in Evolve Cycles

During Evolve cycles, the Conductor reviews recent Success Library entries to:
1. Identify patterns that should be formalized into the Code Constitution or Convention Map
2. Reinforce prompt patterns that produced good adaptations
3. Balance the Evolve agenda — not all Evolve work is remediation; some is amplification of what works
